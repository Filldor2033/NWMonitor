## 📖 Описание проекта

**NWMonitor** — система мониторинга компьютеров в аудитории колледжа. Центральный Flask-сервер отображает состояние каждого ПК на визуальной схеме в реальном времени: через ICMP-пинг и heartbeat-сигналы от клиентского агента на студенческих машинах.

### 🎯 Цели проекта

- **Наглядность** — визуальная схема аудитории (ряды → парты → ПК) с цветовой индикацией
- **Двойной контроль** — heartbeat от клиента + независимый ping-мониторинг
- **Минимум настройки** — клиент разворачивается wizard'ом за одну минуту
- **Надёжность** — WAL-режим SQLite, асинхронный пинг, защита от race condition
- **Расширяемость** — модульная архитектура, Admin Mode с меню действий

### 🔄 Поток данных

1. **Клиент запускается** → `student_client.py` читает `client_config.json`, запускает HTTP-сервер управления
2. **Heartbeat** → каждые 30 секунд клиент отправляет `POST /api/client/heartbeat` с IP, именем и координатами места
3. **Ping-чек** → сервер каждые 30 секунд независимо пингует все устройства (asyncio + ping3)
4. **Статус** → браузер каждые 30 секунд опрашивает `GET /api/device_statuses` и обновляет карточки без перезагрузки
5. **Admin** → администратор удаляет устройство с места; сервер отправляет команду клиенту на удаление конфига

### 🌟 Ключевые особенности

#### 💓 Двойной контроль состояния ПК

- **Heartbeat-статус** — клиент сам сообщает о своей работе; статус протухает через 2 минуты молчания
- **Ping-статус** — независимая ICMP-проверка каждые 30 секунд через asyncio
- **Combined-статус** — итоговый приоритет: `heartbeat` → `ping` → `offline`

#### 🗂️ Визуальная схема аудитории

- Карточки ПК организованы по рядам и партам (2 ряда × 4 парты × 2 ПК)
- Цветовая индикация: 🟢 онлайн / 🔴 офлайн / ⚫ нет данных
- Назначение устройства на место — прямо через веб-интерфейс

#### 👑 Admin Mode

- Включается кнопкой на главной странице
- На каждом занятом ПК появляется кнопка **⋯** с выпадающим меню
- Удаление с места отправляет HTTP-команду клиенту на удаление `client_config.json`

#### ⚡ Автообновление без перезагрузки

- JavaScript-поллинг каждые 30 секунд
- Обновляет только изменившиеся элементы DOM (карточки и пилюли в таблице)

### 📊 Статусы устройства

| Статус            | Условие                               | Цвет |
| ----------------- | ------------------------------------- | ---- |
| **Онлайн**        | Heartbeat получен менее 2 минут назад | 🟢    |
| **Онлайн (пинг)** | Heartbeat устарел, но пинг проходит   | 🟢    |
| **Офлайн**        | Ни heartbeat, ни успешного пинга      | 🔴    |
| **Нет данных**    | Устройство ещё ни разу не проверялось | ⚫    |

## 🚀 Быстрый старт

### 📋 Предварительные требования

- **Python** `3.10` или выше
- **pip** (менеджер пакетов Python)
- **Git** (для клонирования репозитория)
- На Linux — права на ICMP: `sudo setcap cap_net_raw+ep $(which python3)`

### 🔽 Установка сервера

#### 1️⃣ Клонирование репозитория

```bash
git clone https://github.com/Filldor2033/nwmonitor.git
cd NWMonitor
```

#### 2️⃣ Создание виртуального окружения

**Linux / macOS:**

```bash
python -m venv venv
source venv/bin/activate
```

**Windows:**

```bash
python -m venv venv
venv\Scripts\activate
```

#### 3️⃣ Установка зависимостей

```bash
pip install -r requirements.txt
```

#### 4️⃣ Запуск сервера

```bash
python app.py
```

#### 5️⃣ Открытие в браузере

Перейдите по адресу: **[http://localhost:5000](http://localhost:5000/)**

------

### 🖥️ Установка клиента на студенческий ПК

#### 1️⃣ Скопировать `student_client.py` на ПК, установить зависимости

```bash
pip install requests
```

#### 2️⃣ Первичная настройка

```bash
python student_client.py --setup
```

Wizard задаст вопросы: URL сервера, аудитория, ряд, парта, номер ПК, интервал, порт управления.

#### 3️⃣ Проверка (однократный heartbeat)

```bash
python student_client.py --once
```

#### 4️⃣ Постоянная работа

```bash
python student_client.py
```

> 💡 Для автозапуска на Windows добавьте скрипт в Планировщик задач или `shell:startup`.
>  На Linux — оформите как `systemd`-сервис.

## 📖 Использование

### 🗺️ Схема аудитории

| Действие                 | Описание                                                     |
| ------------------------ | ------------------------------------------------------------ |
| **Просмотр статуса**     | Карточки ПК окрашены по статусу: зелёный / красный / серый   |
| **Выбор аудитории**      | Переключатель в верхней части страницы                       |
| **Назначить устройство** | Кнопка «Назначить» в таблице — укажите аудиторию, ряд, парту, ПК |
| **Автообновление**       | Статусы обновляются каждые 30 секунд без перезагрузки страницы |

### 🖥️ Формат размещения

| Поле      | Описание    | Значения                  |
| --------- | ----------- | ------------------------- |
| `room`    | Аудитория   | Например: `3`             |
| `row_no`  | Ряд         | `1` или `2`               |
| `desk_no` | Парта       | `1` — `4`                 |
| `pc_no`   | ПК на парте | `1` — левый, `2` — правый |

### 👑 Admin Mode

После включения Admin Mode на каждой карточке занятого ПК появляется кнопка **⋯**:

| Пункт меню                   | Действие                                                     |
| ---------------------------- | ------------------------------------------------------------ |
| 🖥️ **Управление компьютером** | В разработке                                                 |
| ✕ **Удалить с места**        | Снимает устройство с места в БД и отправляет клиенту команду удалить `client_config.json` |

> ⚠️ Для выполнения удалённой команды клиент должен быть запущен и доступен по `client_control_port`.

## 🔌 API-эндпоинты

### 📡 Клиентский API

| Метод  | Эндпоинт                | Описание                                                    |
| ------ | ----------------------- | ----------------------------------------------------------- |
| `POST` | `/api/client/heartbeat` | Принять heartbeat от клиента, обновить статус устройства    |
| `GET`  | `/api/device_statuses`  | JSON со статусами всех устройств (для браузерного поллинга) |

### 🗺️ Веб-интерфейс

| Метод  | Эндпоинт                        | Описание                             |
| ------ | ------------------------------- | ------------------------------------ |
| `GET`  | `/`                             | Главная страница со схемой аудитории |
| `POST` | `/assign/<id>`                  | Назначить устройство на место        |
| `POST` | `/admin_mode`                   | Включить / выключить Admin Mode      |
| `POST` | `/admin/remove_assignment/<id>` | Удалить устройство с места *(Admin)* |

### 📊 Мониторинг

| Метод | Эндпоинт        | Описание                                  |
| ----- | --------------- | ----------------------------------------- |
| `GET` | `/refresh_ping` | Запустить внеплановый ping-чек            |
| `GET` | `/ping_status`  | Прогресс текущего ping-чека               |
| `GET` | `/scan_status`  | Статус полного сканирования               |
| `GET` | `/export/<id>`  | Экспорт истории проверок устройства в CSV |

### 🖥️ Admin-команды на клиенте

| Метод  | Эндпоинт (клиент, порт 8765) | Описание                                     |
| ------ | ---------------------------- | -------------------------------------------- |
| `POST` | `/admin/delete_config`       | Удалить `client_config.json` (требует токен) |

## 🧪 Тестирование

### Запуск тестов

```bash

pip install pytest pytest-cov

# Запуск всех тестов
pytest tests/ -v
```

| ------------------------- | ------------------------------------ |
| `tests/test_nwmonitor.py` | Полный набор тестов: БД, API, клиент |

### Покрываемые сценарии

- ✅ Инициализация БД и автомиграции
- ✅ Идемпотентность `init_db()`
- ✅ Добавление недостающих колонок в старую БД
- ✅ Регистрация устройства через heartbeat

- ✅ Эндпоинт `/api/device_statuses` — структура и статусы
- ✅ Назначение устройства на место (`/assign`)
- ✅ Экспорт CSV (`/export`)
