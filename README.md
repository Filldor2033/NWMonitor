# NWMonitor

Система мониторинга компьютеров в аудитории колледжа.

## Что умеет

- Центральный сервер (текущий `app.py`) показывает состояние ПК.
- Визуализация аудитории: 2 ряда, 4 парты в ряду, 2 ПК на парте.
- Статус каждого ПК определяется пингом (`online/offline`).
- Клиент на студенческом ПК отправляет heartbeat и своё место в аудитории.
- Для любого устройства можно вручную назначить место в веб-интерфейсе.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск сервера

```bash
python app.py
```

Откройте в браузере:

```text
http://<server-ip>:5000/
```

## Клиент на студенческом ПК

На каждом ПК запускается `student_client.py`.

1. Первый запуск с настройкой:

```bash
python student_client.py --setup
```

2. Проверить отправку одного heartbeat:

```bash
python student_client.py --once
```

3. Обычный режим (постоянная отправка):

```bash
python student_client.py
```

Клиент сохраняет конфиг в `client_config.json` рядом со скриптом.

## Формат места

- `room` - аудитория (например `3`)
- `row_no` - ряд (`1` или `2`)
- `desk_no` - парта (`1..4`)
- `pc_no` - компьютер (`1` слева, `2` справа)

## Admin Mode

- На главной странице есть переключатель `Включить Admin Mode`.
- В этом режиме на схеме аудитории появляются кнопки `Удалить с места`.
- При удалении:
  - устройство снимается с места в серверной БД;
  - сервер отправляет на клиент команду удалить `client_config.json`.
- Для удалённой команды клиент должен быть запущен и доступен по своему `client_control_port`.
